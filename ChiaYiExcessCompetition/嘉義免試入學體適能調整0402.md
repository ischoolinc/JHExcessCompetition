# 嘉義免試入學體適能調整（2026-04-02）

## 1. 修改原因

- 原先 `sit_up`（一分鐘屈膝仰臥起坐）與 `curl`（仰臥捲腹）被當成兩項各自計分。
- 兩者皆達標時會肌耐力類重複加分（+6）或 `addCount` 重複累加（+2），不符合規定「肌力及肌耐力取最高：仰臥捲腹 **或** 一分鐘屈膝仰臥起坐」同類只算一次。

## 2. 修改檔案

| 檔案 | 說明 |
|------|------|
| `DAO/StudentInfo.cs` | `CalcFitnessScore()`：肌耐力改為 `sit_up OR curl` 只加 3 分一次 |
| `DAO/rptStudentInfo.cs` | `CalcFitnessInfoList()`：與主程式一致 |
| `DAO/QueryData.cs` | `FillStudentFitness` 兩處（`StudentInfo` DataRow、`rptStudentInfo` 成績冊）：`isAddFitnessScore` 用的 `addCount` 對肌耐力只算一次 |

## 3. 修改內容

- 以 `sitUpPass`／`curlPass`（或 `sitUpAdd`／`curlAdd`）分別判斷是否達標。
- 肌耐力類僅在 `sitUpPass || curlPass`（或銅牌以上計數）時執行一次 `score += 3` 或 `addCount++`。
- 柔軟度、瞬發力、心肺耐力（`cardiorespiratory`／`pacer`）邏輯未改。

## 4. 不變的規則

- `sit_and_reach`、`standing_long_jump` 邏輯不變。
- 心肺耐力同類只算一次之邏輯不變。
- `addCount >= 4`、`isAddFitnessScore`、總分封頂 9 分（再加條件 +1）之設計不變。

## 5. 測試建議（手動／案例）

| 案例 | 預期 |
|------|------|
| 僅 sit_up 達標（中等以上） | 肌耐力 +3；`addCount` 該類 +1 |
| 僅 curl 達標 | 同上 |
| sit_up 與 curl 皆達標 | 肌耐力仍只 +3；`addCount` 該類仍只 +1 |
| 兩者皆未達標 | 不因此加分／不因此增加該類計數 |
| 主程式與成績冊 | `CalcFitnessScore` 與 `CalcFitnessInfoList` 分數一致 |
