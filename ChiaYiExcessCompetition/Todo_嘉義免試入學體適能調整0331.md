
# 🎯 目標

調整嘉義免試入學體適能計算邏輯，修正 `cardiorespiratory_degree` 與 `pacer_degree` 的重複計算問題：

**新規則：**

* `cardiorespiratory_degree`、`pacer_degree` 屬於同一類「心肺耐力」項目
* 當兩者任一達標時，只能算 **1 次加 3 分**
* 當兩者都達標時，**仍只能加 3 分，不能變成加 6 分**
* 在 `isAddFitnessScore` / `addCount` 判斷時，這兩者也只能算 **1 項達標**
* 其他原本邏輯不要更動
* 修改完成後記錄在：`嘉義免試入學體適能調整0331.md`

## 3.3 將 cardiorespiratory / pacer 改為同一組判斷

### 目標

把：

* `cardiorespiratory_degreeList`
* `pacer_degreeList`

視為同一組「心肺耐力類」項目。

### 規則

* 任一達標：加 3 分
* 兩者都達標：仍只加 3 分
* `addCount` 也只能加 1 次

---

## 3.4 建議實作方式

請先各自判斷是否達標：

```csharp
bool cardiorespiratoryPass = false;
foreach (string name in passStringList)
{
    if (cardiorespiratory_degreeList.Contains(name))
    {
        cardiorespiratoryPass = true;
        break;
    }
}

bool pacerPass = false;
foreach (string name in passStringList)
{
    if (pacer_degreeList.Contains(name))
    {
        pacerPass = true;
        break;
    }
}
```

然後改成「同組只算一次」：

```csharp
// 心肺耐力類：cardiorespiratory / pacer 只算一次
if (cardiorespiratoryPass || pacerPass)
{
    score += 3;
    addCount++;
}
```

---

## 3.5 不要保留這種寫法

不要變成以下邏輯：

```csharp
if (cardiorespiratoryPass)
{
    score += 3;
    addCount++;
}

if (pacerPass)
{
    score += 3;
    addCount++;
}
```

因為這會造成：

* `score` 可能加 6 分
* `addCount` 可能加 2 次

這正是本次要修正的問題。
