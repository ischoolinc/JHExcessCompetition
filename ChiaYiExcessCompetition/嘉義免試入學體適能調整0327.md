# 嘉義免試入學體適能調整（0327）

## 7.1 修改原因

- 先前已將 `curl`（仰臥捲腹）、`pacer`（漸速耐力跑）納入體適能分數計算（每項達標可計入 +3 分，總分仍封頂 9 分）。
- `isAddFitnessScore`（銅牌以上再加 1 分）原先僅依**坐姿體前彎、立定跳遠、仰臥起坐、心肺耐力**四項是否皆達銅牌以上（`addCount == 4`）判斷，`curl`／`pacer` 未參與此計數。
- 依需求改為：**六項**（上述四項 + `curl` + `pacer`）中，**至少四項**達銅牌以上即 `isAddFitnessScore = true`；門檻仍為「至少 4 項」，改以 `addCount >= 4` 表達（避免五、六項皆達標時無法觸發）。

## 7.2 修改檔案

| 檔案 | 說明 |
|------|------|
| `DAO/QueryData.cs` | `FillStudentFitness`：`curl_degree`、`pacer_degree` 與原四項相同，以 `addStringList`（金牌／銀牌／銅牌）累加 `addCount`；條件改為 `addCount >= 4`。`FillRptFitnessInfo`：成績冊路徑同步納入 `curl`／`pacer` 之 `addCount` 與 `>= 4`。 |
| `DAO/rptStudentInfo.cs` | 新增 `curl_degreeList`、`pacer_degreeList`；`CalcFitnessInfoList` 補上與 `StudentInfo.CalcFitnessScore` 一致之 curl／pacer 計分迴圈，避免成績冊與免試邏輯不一致。 |
| `DAO/StudentInfo.cs` | 僅更新 `isAddFitnessScore` 之 XML 註解，說明實際於 `FillStudentFitness` 設定（邏輯未改在 `CalcFitnessScore` 內）。 |

## 7.3 修改內容

- 將 `curl_degree`、`pacer_degree` 納入與原四項**相同**之達標判斷（`addStringList.Contains(等第)`）並累加 `addCount`。
- 觸發條件：`if (addCount >= 4) isAddFitnessScore = true`。
- `StudentInfo.CalcFitnessScore()` 內每項 +3、封頂 9、再依 `isAddFitnessScore` +1 之流程**未改**。

## 7.4 不變的規則

- 每項達標之 +3 分與 `passStringList`（金牌／銀牌／銅牌／中等／免測）邏輯不變。
- 總分上限 9 分（加分前）不變；`isAddFitnessScore` 仍為額外 +1 分。
- 資料讀取欄位（`curl`、`curl_degree`、`pacer`、`pacer_degree`）與 `FillStudentFitness` 之 SQL 未再變更。

## 7.5 測試結果

以下為依規格可執行之**情境檢核**（未於此環境執行自動化測試；請於測試庫以實際學生資料驗證）。

| 情境 | 預期 |
|------|------|
| 原四項皆銅牌以上 | `addCount >= 4`，`isAddFitnessScore = true` |
| 原四項僅 3 項銅牌以上，但 curl 或 pacer 其中一項銅牌以上 | `addCount = 4`，`isAddFitnessScore = true` |
| 六項中僅 3 項銅牌以上 | `isAddFitnessScore = false` |
| 六項中 5 或 6 項銅牌以上 | `isAddFitnessScore = true`；仍僅多 +1 分（非重複加），且 `score > 9` 仍會封頂為 9 再加 1 之設計與既有程式一致 |
| curl／pacer 空白或「未檢測」 | 不計入該項達標、不拋例外；不影響其餘項判斷 |
